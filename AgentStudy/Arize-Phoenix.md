# Arize Phoenix

Arize Phoenix는 AI 애플리케이션의 디버깅, 평가 및 개선을 위한 **오픈 소스 관측성(Observability) 및 평가 플랫폼**입니다.
OpenTelemetry(OTLP) 및 OpenInference 계측 기술을 기반으로 구축되었으며, AI 엔지니어가 애플리케이션의 내부 동작을 투명하게 파악할 수 있도록 돕습니다.

---

## 핵심 개념

### OpenTelemetry & OpenInference

Phoenix는 **OpenTelemetry(OTel)** 표준을 기반으로 데이터를 수집합니다.
OpenTelemetry는 분산 시스템의 관측성 데이터(Trace, Metric, Log)를 수집하기 위한 업계 표준 프레임워크이며,
Phoenix는 여기에 AI/LLM 특화 시맨틱을 정의한 **OpenInference** 규격을 추가로 사용합니다.

### Span

추적의 가장 기본 단위입니다. 하나의 작업(LLM 호출, 도구 실행, 문서 검색 등)을 나타내며, 아래 정보를 포함합니다.

| 항목               | 설명                                              |
| ------------------ | ------------------------------------------------- |
| `span_kind`        | 작업 유형 (LLM, Retriever, Tool, Chain, Agent 등) |
| `input` / `output` | 해당 작업의 입력값과 출력값                       |
| `latency`          | 작업 소요 시간                                    |
| `status`           | 성공 / 오류 여부                                  |
| `attributes`       | 토큰 수, 모델명, 임베딩 벡터 등 추가 메타데이터   |

### Trace

하나의 요청(Request) 처리 흐름 전체를 나타냅니다.
여러 Span이 부모-자식 관계로 연결되어 트리 구조를 이루며, 이를 통해 전체 실행 경로를 한눈에 파악할 수 있습니다.

```
Trace
└── Agent Span (루트)
    ├── Retriever Span (문서 검색)
    │   └── Embedding Span (쿼리 임베딩)
    └── LLM Span (응답 생성)
        └── Tool Span (도구 호출)
```

### Project

관련 Trace들을 묶어서 관리하는 단위입니다.
예를 들어 "RAG 챗봇 v1", "고객 지원 에이전트" 처럼 서비스 또는 실험 단위로 구분하여 Trace를 분리 관리할 수 있습니다.

## 주요 기능

### 1. 추적 (Tracing)

LLM 호출, 데이터 검색(Retrieval), 도구 사용 및 사용자 정의 로직 등 실행 과정을 단계별로 시각화합니다.
실행 시간(Latency)이 어디에서 많이 소요되는지, 비용이 얼마나 발생하는지 등을 실시간으로 디버깅할 수 있습니다.

### 2. 평가 (Evaluation)

LLM 기반 평가자(LLM-as-a-judge), 코드 기반 검사, 또는 사람의 수동 레이블링을 통해 애플리케이션의 출력 품질을 측정합니다.
독성(Toxicity), 답변 관련성, 사실성(Faithfulness) 등 다양한 사전 빌드된 평가 템플릿을 제공합니다.

### 3. 프롬프트 엔지니어링 (Prompt Engineering)

프롬프트 버전을 관리하고, Prompt Playground에서 실시간 프로덕션 예시를 사용해 여러 프롬프트와 모델 변형을 나란히 테스트할 수 있습니다.
과거 호출을 재현(Span Replay)하여 프롬프트 변경이 출력에 미치는 영향을 즉시 확인할 수 있습니다.

### 4. 데이터셋 및 실험 (Datasets & Experiments)

추적 데이터를 데이터셋으로 그룹화하고, 동일한 입력에 대해 애플리케이션의 다른 버전을 실행하여 성능 개선 여부를 체계적으로 비교 분석할 수 있습니다.

## 주요 장점

| 장점                         | 설명                                                                                                                                         |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **완전 무료 & 오픈 소스**    | LangSmith 같은 폐쇄형 대안과 달리 소스 코드가 공개되어 있어 투명성이 높고, 누구나 자유롭게 수정 및 통합 가능                                 |
| **무료 자체 호스팅**         | 라이선스 비용이나 기능 제한 없이 사용자의 인프라에 직접 설치하여 사용 가능                                                                   |
| **강력한 데이터 프라이버시** | 모든 추적 데이터와 프롬프트가 사용자 인프라 내에 머물며 외부로 전송되지 않아 보안이 중요한 기업 환경에 적합                                  |
| **프레임워크 비종속성**      | LangChain, LlamaIndex, CrewAI, DSPy 등 주요 오케스트레이션 도구 및 사용자 정의 에이전트와 호환. Python, TypeScript, Java 등 다양한 언어 지원 |
| **쉬운 연동**                | `phoenix.otel` 래퍼를 통해 코드 변경을 최소화하면서 자동 계측(Auto-instrumentation) 설정 가능                                                |
| **Arize AX로의 확장성**      | 대시보드, 전담 지원 등 더 많은 기능이 필요한 경우 엔터프라이즈 제품인 Arize AX로 업그레이드 경로 제공                                        |

## Phoenix vs Arize AX

Phoenix는 오픈 소스 버전, **Arize AX**는 Arize AI가 제공하는 상용 엔터프라이즈 버전입니다.
Phoenix로 시작하고, 조직의 규모나 요구사항이 커지면 Arize AX로 전환하는 경로를 공식적으로 지원합니다.

| 항목 | Phoenix (오픈 소스) | Arize AX (엔터프라이즈) |
|------|-------------------|------------------------|
| **비용** | 완전 무료 | 유료 (기업 계약) |
| **호스팅** | 자체 호스팅 (Self-hosted) | 클라우드 관리형 (SaaS) 또는 자체 호스팅 |
| **데이터 보관** | 사용자 인프라 내 | Arize 클라우드 또는 사용자 인프라 |
| **추적 & 평가** | 지원 | 지원 |
| **대시보드** | 기본 UI | 고급 커스텀 대시보드 |
| **알림 / 모니터링** | 미지원 | 지원 (드리프트 감지, 임계값 알림 등) |
| **RBAC (역할 기반 접근제어)** | 미지원 | 지원 |
| **SSO / 엔터프라이즈 인증** | 미지원 | 지원 |
| **SLA / 전담 지원** | 커뮤니티 지원 | 전담 기술 지원 및 SLA 보장 |
| **확장성** | 소규모~중규모 | 대규모 프로덕션 환경 |

### 선택 가이드

- **Phoenix 추천**: 개인 프로젝트, 스타트업 초기, 데이터 프라이버시가 최우선인 경우, 비용을 최소화하고 싶은 경우
- **Arize AX 추천**: 팀 단위 협업, 대규모 프로덕션 모니터링, 고급 알림/대시보드가 필요한 경우, 엔터프라이즈 보안 요건(SSO, RBAC)이 있는 경우
