# '창고'가 없는 '전문 배달원'

기존의 프로메테우스 서버가 데이터를 수집해서 자기 창고(저장소)에 쌓아두고 분석까지 하는 **거대한 물류센터**라면, 프로메테우스 에이전트는 오직 데이터를 수집해서 중앙으로 보내기만 하는 **전문 배달원**입니다.

1. 왜 쓸까요? (가장 큰 장점)
   - 다이어트한 프로메테우스: 데이터를 저장하고 분석하는 복잡한 기능을 모두 뺐습니다. 덕분에 컴퓨터 자원(CPU, 메모리)을 훨씬 적게 차지해서 아주 가볍게 돌아갑니다.

   - 에지(Edge) 환경의 강자: 사양인 낮은 작은 기기나, 수천 대의 서버가 있는 복잡한 환경에서 '수집 전용'으로 설치하기에 안성맞춤입니다.

   - 임시 보관 주머니 (WAL): 데이터를 보낼 목적지(중앙 서버)가 잠시 고장 나더라도, 약 2시간 정도는 자신의 주머니(WAL)에 데이터를 담아두었다가 연결이 복구되면 다시 보내주는 끈기를 가지고 있습니다.

2. 무엇을 못 하나요? (포기한 기능)  
   가벼워지기 위해 몇 가지 중요한 기능을 포기했습니다.
   - 직접 조회 불가: 에이전트에게 "지금 수집한 데이터 좀 보여줘"라고 물어볼 수 없습니다. 데이터는 중앙 서버에 가서 확인해야 합니다.

   - 스스로 알람 못 보냄: "서버가 이상해!"라고 직접 소리치는 기능이 없습니다. 이 역할도 데이터를 받은 중앙 시스템이 대신해 줍니다.

3. 주요 장점 및 비즈니스 가치  
   ● 리소스 효율성 및 비용 절감
   - 에이전트 모드는 인덱싱 및 장기 저장 과정을 생략하므로 서버 모드 대비 극히 적은 리소스를 사용합니다

   ● 리소스 산정 기준
   - 일반적으로 수집되는 메트릭 100만 개당 약 1 CPU 코어와 1GB의 메모리가 소요되는 선형적 확장성을 보입니다.

   ● 효율성 비교
   - vmagent와 같은 특화된 에이전트보다는 메모리 사용량이 높을 수 있으나, 표준 프로메테우스 바이너리와 동일한 설정 및 라이브러리를 사용한다는 강력한 호환성 이점이 있습니다.

   ● Egress 전용 통신
   - 중앙 서버가 각 기기에 접속하는 대신, 기기 내부의 에이전트가 중앙으로
     데이터를 밀어넣는(Push) 방식이므로 보안상의 인바운드 트래픽 제한 문제를 해결합니다.

   ● 쿠버네티스와의 연동 좋음
